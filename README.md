# Система модерации объявлений Авито - Frontend

## Технические требования

### Обязательные:
- **Node.js v20+**
- **React v18+**
- **react-router-dom** для роутинга
- **Использование готового API** из папки `tech-int3-server`

## Установка и запуск

### Локальный запуск

1. Установите зависимости:
```bash
npm install
```

2. Запустите бэкенд (из папки `tech-int3-server`):
```bash
cd ../tech-int3-server
npm install
npm start
```

3. Запустите фронтенд:
```bash
npm run dev
```

Приложение доступно на `http://localhost:3000`

### Запуск через Docker Compose

Из корневой папки проекта:
```bash
docker-compose up -d --build
```

Остановка:
```bash
docker-compose down
```

## Команды

- `npm run dev` - запуск dev-сервера
- `npm run build` - сборка продакшн
- `npm run type-check` - проверка типов TS
- `npm test` - запуск тестов
- `npm run test:ui` - запуск тестов с UI

## Обоснование выбора технологий

### TypeScript
Статическая типизация для раннего обнаружения ошибок, улучшения автодополнения и упрощения рефакторинга.

### Ant Design
Готовая дизайн-система с полным набором компонентов, поддержкой темной темы и русской локализацией. Экономит время на разработку UI.

### Vite
Быстрая сборка и HMR благодаря esbuild. Простая конфигурация и поддержка TS из коробки.

### Axios
Удобный API для HTTP-запросов, встроенные интерсепшены для обработки ошибок, поддержка AbortController.

### Vitest
Быстрый тестовый фреймворк на базе Vite, совместимый с Jest API, с поддержкой TS и UI.

### jsPDF
Генерация PDF-отчетов для экспорта статистики. jspdf-autotable для создания таблиц в PDF.

### Docker
Упрощает развертывание, изолирует окружения, обеспечивает консистентность между разработчиками.

## Уточнения
- По логике сортировка может быть по всем 3-м пунктам, однако бэк предполагает только по 1-му (сделала по одному)
- В ТЗ не написано, что нужно указывать "Объявление №: ...", однако в бэке указывается id объявлнения, то есть предполагается его написание (сделала с привязкой к бэку)
- Изображения не притягиваются с бэка, есть URL, но самих картинок нет (перешла по ссылкам - тоже нет, использовала плейсхолдеры)
- В статистике за 7 дней / 30 дней значения проверки = 0 исходя из бэка (я думаю так и надо)
- При F12 могут быть ошибки запросов, так как это нормально в dev-режиме из-за React StrictMode, AbortController и быстрой смены страниц/параметров
- console.error вместо системы логирования, нет валидации данных от API и только базовые тесты (так как это тестовое задание, а не для продакшена)
